Build a Multi-Tenant Roles & Permissions System (RBAC)

IMPORTANT:

This module is part of a centralized multi-tenant SaaS CRM platform for barber shops and hair salons.

The platform architecture behaves similarly to:

- Shopify
- Fresha
- Vagaro
- enterprise SaaS platforms

The system must implement:

- authentication
- authorization
- tenant isolation
- role-based access control (RBAC)
- granular permissions

==================================================
CORE ACCESS CONTROL CONCEPT

The platform has ONE centralized CRM.

There are 3 main roles:

1. super_admin
2. owner
3. staff

IMPORTANT:

The system must enforce strict tenant isolation.

Only:

- super_admin can access the full platform
- owner/staff users only access their assigned tenant/business

==================================================
ROLE DEFINITIONS

---

1. SUPER ADMIN

---

There is ONLY ONE super_admin role.

The super_admin controls:

- the entire SaaS platform
- all tenants
- all users
- all analytics
- all memberships
- all services
- platform settings

The super_admin can:

- create tenants
- edit tenants
- suspend tenants
- create owner users
- create staff users
- assign memberships
- assign services
- view all analytics
- access audit logs
- manage global settings

IMPORTANT:
super_admin bypasses tenant restrictions.

==================================================
2. OWNER ROLE

The owner role belongs to a specific tenant/business.

IMPORTANT:
An owner MUST be assigned to:

- business_id

The owner ONLY manages:

- their own tenant
- their own website
- their own customers
- their own appointments
- their own memberships
- their own staff

The owner cannot:

- access other tenants
- access global CRM
- access super_admin routes

==================================================
OWNER FEATURES

Owners can manage:

- website CMS
- services visibility
- memberships visibility
- appointments
- schedules
- business hours
- customers
- analytics
- staff users

==================================================
3. STAFF ROLE

The staff role belongs to a specific tenant/business.

IMPORTANT:
A staff user MUST be assigned to:

- business_id

Staff users have LIMITED permissions.

The staff ONLY accesses:

- allowed modules
- allowed sections
- assigned functionality

==================================================
GRANULAR PERMISSIONS SYSTEM

The platform must support modular permissions.

Permissions should be configurable per role/user.

Examples of sections/modules:

- dashboard
- appointments
- customers
- services
- memberships
- analytics
- schedules
- website_cms
- payments
- staff_management

==================================================
PERMISSION ASSIGNMENT

When creating/editing a role or staff user:

The owner/super_admin can select:

- which sections the user can access
- which actions are allowed

Example permissions:

- view
- create
- edit
- delete

==================================================
EXAMPLE STAFF CONFIGURATION

Example:
Staff member:

- can view appointments
- can create appointments
- cannot delete appointments
- cannot access analytics
- cannot access memberships
- cannot access billing

==================================================
ROLE ASSIGNMENT FLOW

SUPER ADMIN FLOW:

1. Create tenant/business
2. Create owner user
3. Assign owner role
4. Assign business_id

OWNER FLOW:

1. Create staff user
2. Assign staff role
3. Select permissions/modules
4. Assign business_id

==================================================
TENANT ISOLATION

IMPORTANT:

ALL queries must validate:

- business_id
- role
- permissions

Owner/staff users MUST NEVER access:

- another tenant's data
- another tenant's customers
- another tenant's appointments
- another tenant's memberships
