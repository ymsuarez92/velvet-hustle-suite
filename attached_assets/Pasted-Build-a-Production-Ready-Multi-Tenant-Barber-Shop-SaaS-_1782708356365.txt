Build a Production-Ready Multi-Tenant Barber Shop SaaS CRM Platform (Replit Version)

IMPORTANT:

This is NOT a simple landing page project.

This is a centralized MULTI-TENANT SaaS CRM platform for barber shops and hair salons.

The platform dynamically manages:

- multiple businesses (tenants)
- public websites/landing pages
- memberships/subscriptions
- customer CRM
- appointments
- schedules
- analytics
- CMS website editing

The architecture should feel similar to:

- Shopify
- Fresha
- Vagaro
- GlossGenius

==================================================
REPLIT REQUIREMENTS

This project MUST be optimized for Replit development and deployment.

Requirements:

- clean architecture
- easy local development
- scalable folder structure
- environment variable support
- production-ready setup
- responsive modern UI
- reusable components

Use:

- modern TypeScript
- modular architecture
- scalable SaaS structure

==================================================
CORE PLATFORM ARCHITECTURE

The system has 3 major layers:

1. Global SaaS CRM Platform
2. Tenant Public Websites
3. Tenant Admin Dashboards

IMPORTANT:

The websites are NOT separate projects.

All websites are dynamically generated from the same centralized SaaS platform and database.

- Super Admin accesses the global SaaS dashboard.

Examples:

- elitebarber.domain.com

- barberb.domain.com

IMPORTANT:

The websites are NOT separate deployments.

==================================================
MEMBERSHIP & PAYMENT FLOW

Integrate Stripe subscriptions.

Flow:

1. Customer selects membership
2. Backend creates Stripe Checkout Session
3. Customer pays securely
4. Stripe sends webhook
5. Backend validates payment
6. Membership activates automatically

Store:

- subscriptions
- recurring payments
- renewals
- invoices
- cancellations

IMPORTANT:
Never trust frontend payment status.
